# Phase16-Step5: 見本市シーン（Showcase）によるマップ品質の最終判定

## 目的

全地形要素を詰め込んだ10×10のジオラマを1枚、アートディレクション込みで完成させ、
「このクオリティで量産に進んで良いか」の go / no-go を判定する。

これはマップアセット量産（14地形タイプへの展開）の前の最終関門であり、
ここで妥協すると量産後に全部作り直しになる。逆にここが通れば、以降の地形追加は
テンプレートに従った機械的な作業になる。

## 前提

- Phase16-Step4まで完了している（本番アセット + 調整済みトーン）
- `samples/DioramaSample.tscn` が存在する（拡張または新規作成の土台にする）
- マップテーマは `assets/terrain/theme_default.tres` を使用する（専用テーマを作らない。
  本シーンの目的は「本番設定のままでの品質判定」のため）

## 作業内容

### 1. ShowcaseMap の作成

`samples/ShowcaseMap.tscn` を作成する。10×10のマップデータに以下をすべて含めること。

地形要素:
- 草原（高さ0〜1、マップの半分程度）
- 土の小道（草原を横切る、幅1マス）
- 石の高台（高さ2〜3、崖3段が1箇所以上できる配置）
- 水辺（2×3マス以上の入江状、草原と接する）
- 溶岩だまり（1×2マス以上、石地形と接する）
- 階段（高台への1段接続、1箇所以上）

装飾（既存propの範囲で良い）:
- grass_patch を草原に5〜8個
- broken_stone を石地形の周辺に2〜3個
- flag_placeholder を高台に1個

構図の意図:
- ゲームカメラの標準角度から見て、1画面に全要素が収まること
- 高低差が画面の奥→手前に下がる配置にし、崖の見せ場を作ること

### 2. ユニットの配置

- 既存のプレースホルダーユニットを3〜4体、草原と高台に配置する
  （マップ品質の判定には「駒が乗った状態」の画が必要なため）

### 3. スクリーンショットの取得

Step4で作ったスクリーンショット機能で以下の3構図を撮影し、`docs/dev/showcase/` に保存する。

1. 標準構図: ゲームプレイ時の角度・ズーム
2. 引き構図: マップ全体が入る俯瞰
3. 寄り構図: 崖と水辺の境界部分のクローズアップ

### 4. 品質判定

以下のチェックリストで判定する。判定は制作者（野村）が行い、
結果と気付きを `docs/dev/phase16-step5-review.md` に記録する。

スタイルの統一感:
- [ ] 全アセットが「同じ世界の物」に見える（テクスチャ密度・色調の不揃いがない）
- [ ] Meshy時代のような「素材の寄せ集め」感がない

目標との距離:
- [ ] タクティクスオウガ/Minecraft Dungeonsを参照イメージとして
      「方向性が同じで、粗いだけ」の状態になっている
      （方向性自体がズレている場合はno-go。粗さは量産しながら改善できる）

ゲームとしての成立:
- [ ] 標準構図でマス目・高低差・地形タイプがすべて判読できる
- [ ] ユニットが背景から浮いて見える（沈んでいない）
- [ ] 30分眺めて「このマップでプレイしたい」と思えるか（主観で良い）

### 5. 判定後のアクション

**go の場合:**
- ShowcaseMapを以後のビジュアル回帰確認用シーンとして固定する
  （アセット追加・調整のたびにこのシーンのスクリーンショットで比較する）
- 残り地形タイプ（砂・雪・毒沼・森 等、design docの14種）の量産計画を立て、
  `texture_request_template.md` ベースの依頼書を地形ごとに起こす
- キャラクターパイプライン（female_base共有リグ検証）へ進む

**no-go の場合:**
- 何が目標と違うかを具体的に言語化する（色/形状/密度/ライト/カメラのどれか）
- 原因が調整で直る場合はStep4に戻る。アセットの作り自体の場合はStep2/3の
  該当アセットを個別修正する。**量産には進まない**

## 完了条件

- ShowcaseMap.tscn が存在し、全地形要素・装飾・ユニットが配置されている
- 3構図のスクリーンショットが保存されている
- phase16-step5-review.md に判定結果と根拠が記録されている
- go判定の場合: 次フェーズ（量産 or キャラクター編）の方針がstatus.mdに反映されている
