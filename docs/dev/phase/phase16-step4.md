# Phase16-Step4: 色味・ライティングの統合調整

## 目的

「灰土の誓い」のダークファンタジーのトーンに向けて、テクスチャパレットと
ライティング/ポストプロセスを**同時に**調整し、マップ全体の空気感を確定させる。

テクスチャの色味とライティングは互いに影響するため、片方ずつ確定させるのではなく
「ライティング設定を先に入れる → その光の下でパレットを調整する」の順で行う。

## 前提

- Phase16-Step3が完了している（地形の見た目要素が一通り本番化されている）
- `docs/lighting.md` （WorldEnvironment実装指示書）が存在する
- レンダラーは GL Compatibility（SSAO/SSIL/ボリューメトリックフォグ/SDFGI不可）
- カメラは正投影 size=14。変更しない
- **盤面の可読性（歩けるマス・高さの段数の判別）が最優先**。雰囲気のために可読性を犠牲にしない

## 作業内容

### 1. ライティングの実装（docs/lighting.md を実施）

- `docs/lighting.md` の指示に従い、Main.tscn の WorldEnvironment に Environment リソースを
  設定し、DirectionalLight を調整する
- 実施後、lighting.md 記載の値を初期値として以下も確認する:
  - Step3の溶岩EMISSIONがグローで発光していること（glow_hdr_thresholdとの整合）
  - フォグが正投影カメラで不自然に見えないこと（depth fogは正投影だと効きが変わるため、
    違和感があれば fog_density を下げるか無効化して adjustments のみで雰囲気を出す）

### 2. スクリーンショット取得の仕組み

調整の反復を回すため、決まった条件でスクリーンショットを撮れるようにする。

- 見本市用マップ（Step5で本格化するが、この時点のサンプルマップで良い）を
  ゲームカメラの標準角度・標準ズームで表示し、F12等のデバッグキーで
  `user://screenshots/` にPNG保存する機能を追加する（デバッグビルド限定で良い）
- 同一構図で撮ることで、調整前後の比較を可能にする

### 3. パレットのトーン調整

**前提の更新（Step2完了時点）**: palette.json は既に灰色トーンへの初期調整が済んでいる
（grass `#3c5c2e` 系、水は暗青緑、溶岩は暗色ベース+発光コア `#ffb238` のみ明色）。
したがって本Stepの作業は「明るいパレットを暗くする」ことではなく、
**暗色ベース同士の判読性の確保と最終トーンの微調整**である。

ライティング適用後のスクリーンショットで以下を重点確認し、必要な調整を行う。

- **grass / dirt / stone の同化リスク**: 3者とも暗色になったため、暗いシーンで
  区別がつくか。つかない場合は明度を離すのではなく**色相方向に離す**
  （grassに緑み、stoneに青みを足す等。明度を上げると世界観が崩れるため）
- **水面の判読**: 暗い水面が影の落ちた地面と誤認されないか。シェーダーの
  スペキュラ/動きで区別できていれば色は変えない
- **溶岩のアクセント機能**: 発光コアがグロー（本Step1で有効化）で光り、
  画面唯一の高彩度要素として機能しているか。EMISSION値との整合を確認
- 全体の一括彩度・明度スケール処理は実装しておく（最終トーンの追い込み用）が、
  初手では使わず、まず地形間コントラストを優先する

調整はパレット全体・地形単位の変更として行い、
個別色の手調整は最後の微調整のみにすること。

### 4. 反復プロトコル

以下のループを回す。**1回の変更項目は1〜2個に絞る**（何が効いたか分からなくなるため）。

1. パレットまたはEnvironment値を変更
2. 再生成 / 再インポート
3. 標準構図でスクリーンショット
4. チェックリスト（下記）で評価、変更を採用/棄却
5. 採用した値と理由を `docs/dev/phase16-step4-log.md` に1行ずつ記録

### 5. 評価チェックリスト

雰囲気:
- [ ] 「明るい牧歌的な世界」ではなく「灰と土の世界」に見える
- [ ] Minecraft Dungeons / タクティクスオウガの持つ陰影の深さに近づいている
- [ ] 溶岩・魔法エフェクト等の発光要素が画面のアクセントとして機能している

可読性（こちらが優先）:
- [ ] 隣接する地形タイプ（草/土/石）がひと目で区別できる
- [ ] 高さの段数が崖の陰影で判別できる
- [ ] 水面と歩行可能マスの区別が明確
- [ ] ユニット（現状のプレースホルダーで良い）が背景に沈まない

## 完了条件

- WorldEnvironment / DirectionalLight が設定され、調整済みの値がMain.tscnに反映されている
- palette.json が調整済みで、全アセットが調整後パレットで再生成されている
- 調整ログ（phase16-step4-log.md）が残っている
- 評価チェックリストが全て通ったスクリーンショットが `docs/dev/` に保存されている

## 備考

- この工程は指示書通りに1回で終わるものではなく、スクリーンショットを見ながらの
  反復作業になる。Claude Codeでの機械的実行と、チャットでの見た目レビューを併用すること
- ユニットの視認性はキャラクター本番化（Phase17以降）で再調整するため、
  ここでは「背景として完成しているか」を判断基準にする

## 実施メモ（1回目セッション）

詳細は `docs/dev/phase16-step4-log.md` を参照。

- ライティング自体（WorldEnvironment / DirectionalLight）はStep4着手前の
  非公式な調整で既に本番値がMain.tscnに入っていた（コミット履歴から再構成）。
  今回はそれを `assets/environment/battle_atmosphere.tres` にリソース化し、
  見本市用マップと共有するようにした（値そのものは変更していない）
- スクリーンショット取得機能（F12、debugビルド限定）を
  `scripts/debug/screenshot_capture.gd` として追加し、
  `samples/Phase16AssetPreview.tscn` に組み込んだ。同シーンは実カメラ
  （`CameraController`）と共有Environmentを使うようにし、Main.tscnと
  同一構図で比較できるようにした
- palette.jsonのstoneを彩度・色相方向に調整（明度は維持）し、対応する
  GLB/テクスチャを再生成した。grass/dirtは現状で色相分離が十分と判断し
  未変更
- **このセッションではゲームの実機起動・スクリーンショット目視確認は
  行っていない**（機械的実装のみ）。次回はユーザーが実機でF12撮影した
  スクリーンショットを共有し、それを見ながらチャットで反復調整する

## 実施メモ（2回目セッション）

詳細は `docs/dev/phase16-step4-log.md` を参照。

- `screenshot_capture.gd`に`--auto-capture`起動オプションを追加し、
  Godotをウィンドウモードで直接起動してClaude Code自身がスクリーンショットを
  撮影・目視レビューできるようにした（`--headless`はdummy描画で使えないため
  不可）
- `voxel_map.gd`の背景プレーンが未調整のプレースホルダー色
  （`#527c3f`、明るい緑）のままで、Environmentの暗い背景色を完全に覆い隠して
  いたことが実際のスクリーンショットで判明。`#313134`に修正。これがMain.tscn
  にも影響する共通コードだったため、最も大きな見た目改善になった
- `ambient_light_energy`を1.2→0.5（lighting.mdの目安上限）に変更。
  1.2/0.7/0.5を比較し、可読性を保ったまま暗さを稼げることを確認
- `Phase16AssetPreview`のサンプルマップにdirt/water/lavaを追加
  （既存のwater配置は崖の死角に隠れて画面に一切映っていなかったため、
  同じ高さの開けた位置に置き直した）
- 完了条件は（ユニット視認性を除き）本セッションで満たした。
  評価スクリーンショットは `docs/dev/phase16-step4-screenshot.png`
