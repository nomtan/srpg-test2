# 環境システム設計ドキュメント（Environment Systems Design）

対象: 灰土の誓い（Ashen Vow）
位置づけ: 地形変化戦闘システムの拡張。マップ上の環境要素（水・溶岩・火災、
将来的に氷・毒沼・瘴気）が時間とともに変化・相互作用する仕組みを統合的に定義する。

本書は `fluid_system_design.md` の内容を統合・置換する（旧文書は破棄して良い）。

ステータス: 流体・火災は設計確定（一部未確定事項あり。§10参照）。
氷・毒沼・瘴気は将来拡張スケッチ（§7）。

---

## 1. アーキテクチャ概要

すべての環境要素は共通の **環境tick基盤（EnvironmentSystem）** の上に実装する。

- **環境tick**: 敵味方問わず**1キャラの行動が終わるごと**に1回実行される更新単位
- **決定性**: 環境の変化に乱数を使わない。同一盤面・同一操作なら結果は常に一致する
  （プレビュー表示・AI評価・リプレイの一致を保証するため）
- **レイヤー構造**: セルは複数の環境状態を同時に持ちうる
  - 地表流体レイヤー: 水 / 溶岩（排他）
  - 地表状態レイヤー: 燃焼中 / 焼け跡 / （将来: 凍結・毒沼）
  - 気体レイヤー: （将来: 瘴気）
- **時間の2単位**:
  - **tick** = キャラ1人の行動終了ごと（拡散・侵食の単位）
  - **ターン** = ラウンド相当の長い単位（持続時間の単位。§10-1で定義を確定すること）

### 環境tickの処理順（固定）

1. 流体の整合性再計算（地形変化があった場合）
2. 流体の後退処理
3. 流体の拡散処理（水3マス → 溶岩1マス）
4. 水源昇格判定
5. 流体相互作用（水×溶岩の石化）
6. 火災: 消火判定（水到達セル）→ 着火判定（溶岩到達・延焼）→ 延焼拡散
7. ユニット影響の解決（ダメージ等）

同一処理内で複数の候補セルがある場合の順序は
「発生源からの距離昇順 → グリッド座標順（y昇順→x昇順）」で固定する。

## 2. 共通ルール: ユニットへの影響

環境状態はユニットに「移動制限」と「ダメージ」の2種類の影響を与える。

| セル状態 | 移動制限 | ダメージ |
|---|---|---|
| 水 | **そのセルから移動開始する場合、最大1マスしか移動できない** | なし |
| 溶岩 | 同上（最大1マス） | **滞在・通過でダメージ** |
| 燃焼中 | 同上（最大1マス） | **滞在・通過でダメージ** |
| 焼け跡 | なし | なし（無害） |

- 「最大1マス」制限は移動開始セルの状態で判定する（泥濘から抜け出すイメージ）
- 「通過」ダメージは移動経路に該当セルが含まれた場合に発生する
- ダメージ量は§11のパラメータで管理する（値は未確定 → §10-2）
- 飛行等のユニット特性による免除はジョブ設計と突き合わせる（§10-5）

## 3. 流体システム（水・溶岩）【確定】

### 3.1 方式と基本パラメータ

- **方式**: Minecraft式の距離減衰方式（高さ充填方式は採用しない）
- 源の強さ: **水 = 6 / 溶岩 = 5**（= 源から届く最大マス数）
- 侵食速度: 1tickあたり **水 = 3マス / 溶岩 = 1マス**（BFSのフロンティアをその層数進める）
- **水源昇格あり / 溶岩源昇格なし**

### 3.2 拡散ルール

- 流体セルは水平隣接（4方向）の**同じ高さ**のセルへ「強さ−1」で広がる。強さ0は広がらない
- 隣接セルが**低い**場合は落下し、**落下先で強さが最大値にリセット**される
- 高いセルへは広がらない
- 流体はセル上面に乗る薄い層として扱い、水位・深さの概念は持たない
  （表示は既存 water_plane / lava_plane の SURFACE_OFFSET 方式）

### 3.3 水源昇格（無限水源）

- 流水セルが**水平隣接する水源セルを2つ以上**持つと水源に昇格する（tick処理内で判定）
- 帰結: 海・湖（全セル水源）に幅2以上の水路を接続すると水源化が連鎖し、
  減衰を超えて広がり続ける。幅1の直線水路は昇格せず6マスで止まる
- → **地形依存のスキル強度**（大水域の近くでは地形変化の影響が劇的になる）。
  マップの水域配置は戦略資源である

### 3.4 溶岩の非対称性

- 溶岩源昇格は存在しない。溶岩は常に有限資源
- 減衰5・1マス/tickの**遅い時限ハザード**。ただし可燃セルに触れると火災を介して
  実効的な脅威範囲が急拡大する（§4）

### 3.5 後退（源の喪失）

- 隆起等で源が失われた場合、依存する流れセルは消滅する
- 消滅は拡散と同レートで末端から後退させる（読みやすさの対称性）。
  実装Phase Aでは即時一括消滅で可

### 3.6 水×溶岩の相互作用（推奨案・要最終確認 → §10-4）

| 接触パターン | 結果 |
|---|---|
| 水が「流れ溶岩」に到達 | セルは**石（stone）地形**に変化、流体消滅 |
| 水が「溶岩源」に到達 | **黒曜石相当の硬質地形**に変化（専用地形を作らない場合はstone） |
| 溶岩が水セルに到達 | 溶岩側の進行セルが石化し進行停止 |

石化セルは歩行可能（「溶岩地帯に水路を引いて足場を作る」戦術の成立）。

## 4. 火災システム【確定】

### 4.1 セル状態遷移

```
通常（可燃propあり） --着火--> 燃焼中（5ターン持続） --燃え尽き--> 焼け跡
```

- **可燃セル** = 可燃フラグを持つprop（草むら・枯れ草・灌木等）が配置されたセル。
  可燃フラグはprop定義側で持つ（§6）
- **燃焼中**: **5ターン持続**（tickではなくターン。定義は§10-1）
- **焼け跡**: 可燃propが消滅し、地表が焦げたテクスチャに変わる。
  戦闘中は永続（プレイヤーの行動が盤面に痕跡として残る）。
  焦げ地表は手続き生成に `scorched` パレットを追加して生成する

### 4.2 着火源

1. **火の魔法**（火系ジョブのスキル。着火対象は可燃セルのみ）
2. **溶岩の侵食が可燃セルに到達した時**（自動連鎖）

### 4.3 延焼

- 燃焼中セルは毎tick、隣接4方向の「可燃propを持つ通常セル」に着火する
- 延焼速度: **3マス/tick**（水の拡散と同じBFS層数）
- 既に燃焼中・焼け跡・不燃のセルには広がらない
- 高低差の扱い: 同じ高さ間でのみ延焼する（初期仕様。斜め・段差越え延焼は入れない）
- 帰結: 火は**燃料で制限**される（流体は距離で制限される、との対比）。
  草地の連なりが火の届く範囲を決め、土・石の帯が天然の防火帯として機能する。
  propの自動散布テーブル（地形タイプ別の密度設定）が実質「可燃マップの設計」を兼ねる

### 4.4 消火

- **水が燃焼セルに到達すると即消火**（そのセルは可燃propを失い、焼け跡になるか
  通常の濡れた地面になるかは §10-3 で確定）
- 環境tick処理順（§1）により、同一tick内では「水の拡散 → 消火 → 着火・延焼」の順で
  解決される（水が先に届けば燃えない）

### 4.5 戦術的位置づけ

火は「安い範囲拒否」。火の魔法1発の直接ダメージが小さくても、草地に撃てば
数ターン持続する侵入抑制エリア（移動1マス制限+ダメージ）を作れる。
一方で自軍の退路も焼き、燃料が尽きれば消える。
水系ジョブは消火によって火系ジョブの直接カウンターになる。

## 5. 相互作用マトリクス（統合）

| ↓が →に到達 | 水 | 溶岩 | 燃焼中 | 可燃prop | 焼け跡 |
|---|---|---|---|---|---|
| **水** | - | 石化（§3.6） | **即消火** | 変化なし | 変化なし |
| **溶岩** | 石化（自分が停止） | - | 変化なし | **着火** | 変化なし |
| **火（延焼）** | 延焼しない | 延焼しない | - | **着火** | 延焼しない |

将来要素（氷・毒沼・瘴気）が入る場合もこの表を拡張する形で定義する。

## 6. propシステムとの連携要件

（prop本番化の指示書側に反映すること）

- prop定義に **可燃フラグ（flammable: bool）** を持たせる
- propは自分の属するセル座標を保持する（燃焼・水没時の表示切り替えに必要）
- 燃焼開始時: 可燃propに炎エフェクトを重ねる（またはprop自体を燃焼見た目に差し替え）
- 燃え尽き時: propを消去し、セル上面テクスチャをscorchedに差し替える
- 水没セルのprop: 非表示にする（流体レイヤーとの連動）

## 7. 将来拡張スケッチ（未確定・実装対象外）

共通基盤に乗せることを前提とした方向性メモ。仕様確定時に本書へ昇格させる。

- **氷（氷系スキル）**: 水セルを凍結させ**歩行可能な床**に変える（期限Nターン）。
  凍結中は流体として振る舞わない（拡散・昇格の対象外）。溶岩隣接や火系スキルで解凍。
  地表状態レイヤーの一種として実装
- **毒沼**: 地表状態レイヤー。滞在・通過で毒付与（DoT）+移動1マス制限。
  地形タイプとして最初から配置される場合と、毒系スキルで生成される場合の両方を想定
- **瘴気（有害ガス）**: **気体レイヤー**（地表と独立した第2層）。
  発生源から減衰付きで拡散し、Nターンで自然消散する。滞在で継続ダメージ or 状態異常。
  地表の流体・火災と同一セルに共存できる点が地表系と異なる。
  風向きによる流されは将来課題。火気（溶岩・燃焼セル）との接触で引火爆発する案は
  面白いが複雑化するため保留

## 8. UI・可読性要件（実装必須）

- **スキルプレビュー**: 地形変化・着火・注水スキルのターゲティング時、
  最終的な影響範囲（流体到達セル・延焼到達セル）を非破壊シミュレーションで半透明表示する
- **次tick予告**: 次の環境tickで変化するセル（流体フロンティア・延焼フロンティア）を
  弱いハイライトで常時表示する
- **残りターン表示**: 燃焼中セルの残り持続ターンをカーソルオーバーで確認できる
- **状態の視認性**: 水/溶岩/燃焼/焼け跡は色・アニメーションだけで区別できること
  （色覚多様性への配慮として形状・動きの差も併用する）

## 9. データ構造・実装指針

```
MapCellData 追加フィールド:
  fluid_type: enum { NONE, WATER, LAVA }
  fluid_is_source: bool
  fluid_strength: int          # 水: 0..6 / 溶岩: 0..5
  surface_state: enum { NONE, BURNING, SCORCHED }   # 将来: FROZEN, POISON
  burning_turns_left: int
  # 将来: gas_type / gas_strength（気体レイヤー）

EnvironmentSystem (新規クラス):
  tick(map_data) -> EnvTickResult          # §1の処理順で1tick分
  recompute_fluids(map_data)               # マルチソースBFS全再計算
  preview(map_data, action) -> 影響セル集合  # 非破壊・収束までシミュレート
  on_turn_boundary(map_data)               # 燃焼持続などターン単位の減算
```

- マップは20×20規模のためBFS全再計算のコストは無視できる。性能最適化は不要
- `preview` はMapDataのコピーに対して実行する（本体を汚さない）
- MapRenderer側: tick結果の差分セルに対しplane/エフェクトの追加・削除を行い、
  BFS順に1セルあたり0.1〜0.2秒ずらした波及アニメーションを付ける
- ターンシステム側に「行動終了フック（→tick）」と「ターン境界フック
  （→燃焼カウント減算）」の2つを設ける

## 12. 実装フェーズ分割

前提: Phase16-Step5（見本市）完了後に着手。prop本番化（可燃フラグ含む）が先行必須。

| Phase | 内容 | 完了条件 |
|---|---|---|
| A: 流体コア | fluidフィールド、BFS再計算、tick拡散（後退は即時）、水源昇格、決定的処理順、移動1マス制限 | デバッグコマンドで陥没→水が3マス/tickで流れ込み、水上ユニットの移動が1に制限される |
| B: 火災コア | surface_state、着火（魔法・溶岩）、延焼、5ターン燃焼、焼け跡テクスチャ、消火、ダメージ | 草地に着火→延焼→燃え尽きの一連と、水による消火が機能する |
| C: 可読性 | スキルプレビュー、次tick予告、波及アニメ、tick後退、残りターン表示 | プレビューと実結果が完全一致する |
| D: 統合 | 水×溶岩石化、AI・経路探索対応、ユニット特性連携 | AIが環境ハザードを回避・利用する |
